# Epic C: 시스템 설정

유저 스토리 ID: US-B-5-01
기능: Feature B-5: [CMS] QuickSight 대시보드 임베딩
제목: QuickSight 대시보드 임베딩
역할: 콘텐트 운영자 (이콘텐)
목표: CMS 내에서 QuickSight 대시보드를 통해 B-1~B-4의 모든 통계를 시각화하여 볼 수 있다
가치: 고급 분석 기능을 별도 도구 없이 CMS에서 바로 사용할 수 있다
우선순위: High

인수 조건 (Acceptance Criteria):
  1. Given CMS 통계 페이지에서, When QuickSight 대시보드 영역을 확인하면, Then 임베딩된 QuickSight 대시보드가 표시된다
  2. Given 대시보드가 로딩될 때, When 인증이 필요하면, Then JWT 토큰 기반으로 자동 인증되어 대시보드가 표시된다
  3. Given 대시보드 내에서, When 필터나 드릴다운을 사용하면, Then QuickSight 기본 인터랙션이 정상 동작한다
  4. Given 대시보드 로딩에 실패했을 때, When 오류가 발생하면, Then 오류 메시지와 재시도 안내가 표시된다

유저 스토리 ID: US-C-1-01
기능: Feature C-1: [CMS] 계정 관리
제목: 관리자 계정 생성
역할: 슈퍼 관리자 (최관리)
목표: 새 관리자 계정을 생성하고 역할(슈퍼 관리자/여론조사 관리자/콘텐트 운영자)을 지정할 수 있다
가치: 적절한 권한을 가진 관리자를 시스템에 추가할 수 있다
우선순위: High

인수 조건 (Acceptance Criteria):
  1. Given 계정 관리 페이지에서, When ID, 비밀번호, 이름, 역할을 입력하고 저장하면, Then 새 관리자 계정이 생성된다
  2. Given 역할 선택 시, When 슈퍼 관리자/여론조사 관리자/콘텐트 운영자 중 하나를 선택하면, Then 해당 역할의 권한이 부여된다
  3. Given 이미 존재하는 ID로 생성을 시도할 때, When 저장하면, Then 중복 ID 오류가 표시된다
  4. Given 필수 입력값이 비어있을 때, When 저장을 시도하면, Then 필수값 입력 안내가 표시된다

유저 스토리 ID: US-C-1-02
기능: Feature C-1: [CMS] 계정 관리
제목: 관리자 계정 조회/수정/비활성화
역할: 슈퍼 관리자 (최관리)
목표: 관리자 계정 목록을 조회하고 정보를 수정하거나 비활성화할 수 있다
가치: 관리자 계정을 관리할 수 있다
우선순위: High

인수 조건 (Acceptance Criteria):
  1. Given 계정 관리 페이지에서, When 계정 목록을 확인하면, Then 모든 관리자 계정이 리스트로 표시된다
  2. Given 계정을 선택했을 때, When 수정 버튼을 클릭하면, Then 이름, 역할 등을 수정할 수 있다
  3. Given 계정을 비활성화할 때, When 비활성화를 실행하면, Then 해당 계정으로 로그인이 불가능해진다
  4. Given 슈퍼 관리자 본인 계정을 비활성화하려 할 때, When 시도하면, Then 자기 자신 비활성화 불가 오류가 표시된다

---

유저 스토리 ID: US-C-2-01
기능: Feature C-2: [CMS] 코드 관리
제목: 공통 코드 CRUD
역할: 슈퍼 관리자 (최관리)
목표: 시스템 공통 코드를 생성, 조회, 수정, 삭제할 수 있다
가치: 시스템에서 사용하는 코드 값을 중앙에서 관리할 수 있다
우선순위: Medium

인수 조건 (Acceptance Criteria):
  1. Given 코드 관리 페이지에서, When 코드 그룹, 코드 값, 코드명, 코드 순서를 입력하고 저장하면, Then 새 공통 코드가 등록된다
  2. Given 코드 목록에서, When 특정 코드를 선택하면, Then 코드 상세 정보(코드 그룹, 코드 값, 코드명, 순서)가 표시된다
  3. Given 코드를 수정했을 때, When 저장하면, Then 변경된 코드 값이 시스템에 반영된다
  4. Given 코드를 삭제할 때, When 삭제를 실행하면, Then 해당 코드가 제거된다
  5. Given 시스템에서 사용 중인 코드를 삭제하려 할 때, When 시도하면, Then 사용 중 삭제 불가 경고가 표시된다

유저 스토리 ID: US-C-3-01
기능: Feature C-3: [CMS] 로그 관리
제목: CMS 접속 로그 조회
역할: 슈퍼 관리자 (최관리)
목표: CMS 접속 로그(로그인/로그아웃 이력)를 조회할 수 있다
가치: 관리자의 시스템 접속 현황을 모니터링할 수 있다
우선순위: Medium

인수 조건 (Acceptance Criteria):
  1. Given 로그 관리 페이지에서, When 접속 로그 탭을 선택하면, Then 관리자별 로그인/로그아웃 이력이 시간순으로 표시된다
  2. Given 접속 로그에서, When 특정 관리자로 필터링하면, Then 해당 관리자의 접속 이력만 표시된다
  3. Given 접속 로그에서, When 기간 필터를 설정하면, Then 해당 기간의 접속 이력만 표시된다
