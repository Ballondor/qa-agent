# Epic A: 여론조사 데이터 시각화 서비스

유저 스토리 ID: US-A-0-01
기능: Feature A-0: [CMS] 선거 관리
제목: 선거 등록
역할: 여론조사 관리자 (박데이터)
목표: 선거 정보(선거명, 선거일, 선거타입)를 등록할 수 있다
가치: 해당 선거에 속하는 정당, 후보자, 여론조사 데이터를 체계적으로 관리할 수 있다
우선순위: High

인수 조건 (Acceptance Criteria):
  1. Given 선거 관리 페이지에서, When 선거명, 선거일, 선거타입(대선/지방선거/국회의원선거 등)을 입력하고 저장하면, Then 선거고유번호가 자동 생성되고 선거가 등록된다
  2. Given 선거명을 입력하지 않았을 때, When 저장을 시도하면, Then 필수값 입력 안내 메시지가 표시된다
  3. Given 선거일을 선택하지 않았을 때, When 저장을 시도하면, Then 필수값 입력 안내 메시지가 표시된다
  4. Given 선거타입을 선택하지 않았을 때, When 저장을 시도하면, Then 필수값 입력 안내 메시지가 표시된다
  5. Given 동일한 선거명이 이미 존재할 때, When 저장을 시도하면, Then 중복 경고가 표시된다
  6. Given 선거가 등록되었을 때, When 선거 리스트를 확인하면, Then 등록된 선거가 목록에 표시된다

유저 스토리 ID: US-A-0-02
기능: Feature A-0: [CMS] 선거 관리
제목: 선거 조회/수정/삭제
역할: 여론조사 관리자 (박데이터)
목표: 등록된 선거 목록을 조회하고 정보를 수정하거나 삭제할 수 있다
가치: 선거 정보를 최신 상태로 유지할 수 있다
우선순위: High

인수 조건 (Acceptance Criteria):
  1. Given 선거 관리 페이지에서, When 선거 목록을 확인하면, Then 등록된 모든 선거가 리스트로 표시된다
  2. Given 선거를 선택했을 때, When 수정 버튼을 클릭하면, Then 선거명, 선거일, 선거타입을 수정할 수 있다
  3. Given 선거를 삭제하려 할 때, When 해당 선거에 연결된 정당/후보자/여론조사 데이터가 존재하면, Then 삭제 불가 경고가 표시된다
  4. Given 하위 데이터가 없는 선거를 삭제할 때, When 삭제 확인을 클릭하면, Then 선거가 삭제된다

유저 스토리 ID: US-A-0-03
기능: Feature A-0: [CMS] 선거 관리
제목: 선거 선택 후 하위 관리
역할: 여론조사 관리자 (박데이터)
목표: 선거를 선택한 후 해당 선거의 정당/후보자/데이터를 관리할 수 있다
가치: 선거별로 분리된 데이터를 관리할 수 있다
우선순위: High

인수 조건 (Acceptance Criteria):
  1. Given 선거 목록에서 특정 선거를 선택했을 때, When 데이터 관리/정당 관리/후보자 관리 메뉴에 진입하면, Then 해당 선거에 속한 데이터만 표시된다
  2. Given 선거가 선택되지 않은 상태에서, When 정당/후보자/데이터 관리에 진입하면, Then 선거 선택 안내 메시지가 표시된다
  3. Given 선거 A에서 등록한 정당이 있을 때, When 선거 B로 전환하면, Then 선거 A의 정당은 표시되지 않는다

유저 스토리 ID: US-A-1-01
기능: Feature A-1: [CMS] 데이터 관리
제목: XLSX 파일 업로드
역할: 여론조사 관리자 (박데이터)
목표: 정해진 양식의 .xlsx 파일을 업로드하고 카테고리를 선택할 수 있다
가치: 여론조사 데이터를 시스템에 등록하여 시각화할 수 있다
우선순위: High

인수 조건 (Acceptance Criteria):
  1. Given 관리자가 데이터 관리 페이지에 접속했을 때, When '정당 지지율' 또는 '후보자 지지율' 카테고리를 선택하고 .xlsx 파일을 업로드하면, Then 파일이 성공적으로 업로드되고 확인 메시지가 표시된다
  2. Given 관리자가 카테고리를 선택하지 않았을 때, When 업로드를 시도하면, Then 카테고리 선택 필수 안내 메시지가 표시된다
  3. Given 관리자가 .xlsx가 아닌 파일을 업로드할 때, When 업로드를 시도하면, Then "파일 형식이 잘못되었습니다." 오류 메시지가 표시된다
  4. Given 파일 업로드 중 네트워크 오류가 발생했을 때, When 업로드가 실패하면, Then 재시도 안내 메시지가 표시된다
  5. Given 빈 파일을 업로드할 때, When 파싱을 시도하면, Then "필수 데이터가 없습니다." 오류 메시지가 표시된다
  6. Given 파일 업로드가 완료되었을 때, When 처리 시간을 측정하면, Then 10초 이내에 완료된다
  7. Given 파일이 업로드되면, When 버전 목록을 확인하면, Then 새 버전으로 등록되어 이력이 관리된다

유저 스토리 ID: US-A-1-02
기능: Feature A-1: [CMS] 데이터 관리
제목: 데이터 유효성 검사
역할: 여론조사 관리자 (박데이터)
목표: 업로드된 파일의 필수 컬럼명, 날짜 형식, 수치 데이터를 자동 검증받을 수 있다
가치: 잘못된 데이터가 시스템에 등록되는 것을 방지할 수 있다
우선순위: High

인수 조건 (Acceptance Criteria):
  1. Given 정당 지지율 파일이 업로드되었을 때, When 유효성 검사가 실행되면, Then 해당 선거에 설정된 필수 컬럼(date, party1~partyN) 존재 여부가 확인된다
  2. Given 후보자 지지율 파일이 업로드되었을 때, When 유효성 검사가 실행되면, Then 해당 선거에 설정된 필수 컬럼(date, candi1~candiN, candi_non, candi_na) 존재 여부가 확인된다
  3. Given 선거별로 필수 컬럼 갯수가 다를 때, When 검사가 실행되면, Then 해당 선거에 등록된 정당/후보자 수에 맞는 컬럼이 검증된다
  4. Given 날짜 컬럼에 잘못된 형식이 있을 때, When 검사가 실행되면, Then "날짜 형식이 잘못되었습니다." 오류 메시지가 표시된다
  5. Given 수치 데이터에 문자열이 포함되어 있을 때, When 검사가 실행되면, Then 해당 셀 위치와 함께 오류가 표시된다
  6. Given 필수 컬럼이 누락되었을 때, When 검사가 실행되면, Then "필수 데이터가 없습니다." 오류 메시지와 누락 컬럼명이 표시된다
  7. Given 수치 데이터가 0~1 범위를 벗어날 때, When 검사가 실행되면, Then 범위 초과 경고가 표시된다
  8. Given 중복된 날짜 데이터가 있을 때, When 검사가 실행되면, Then 중복 날짜 경고가 표시된다
  9. Given 모든 유효성 검사를 통과했을 때, When 결과가 표시되면, Then 검증 성공 메시지와 데이터 요약(행 수, 기간)이 표시된다

유저 스토리 ID: US-A-1-03
기능: Feature A-1: [CMS] 데이터 관리
제목: 데이터 미리보기
역할: 여론조사 관리자 (박데이터)
목표: 업로드된 데이터를 FO와 동일한 화면으로 카테고리별 미리보기할 수 있다
가치: 게시하기 전에 데이터가 올바르게 표시되는지 확인할 수 있다
우선순위: High

인수 조건 (Acceptance Criteria):
  1. Given 파일이 성공적으로 업로드되었을 때, When 미리보기 버튼을 클릭하면, Then FO와 동일한 형태의 그래프가 CMS 내에서 표시된다
  2. Given 정당 지지율 카테고리로 업로드했을 때, When 미리보기를 실행하면, Then 정당 지지율 그래프가 표시된다
  3. Given 후보자 지지율 카테고리로 업로드했을 때, When 미리보기를 실행하면, Then 후보자 지지율 그래프가 표시된다
  4. Given 미리보기 화면에서, When 필터(전체/성별/연령 등)를 변경하면, Then 해당 필터에 맞는 데이터로 그래프가 갱신된다
  5. Given 데이터에 오류가 있을 때, When 미리보기를 시도하면, Then 오류 영역이 dimmed 처리되고 안내 메시지가 표시된다
  6. Given 미리보기 화면에서, When 그래프 렌더링 시간을 측정하면, Then 1초 이내에 완료된다

유저 스토리 ID: US-A-1-04
기능: Feature A-1: [CMS] 데이터 관리
제목: 데이터 버전 관리 및 게시
역할: 여론조사 관리자 (박데이터)
목표: 업로드된 데이터를 버전으로 관리하고, 특정 버전을 선택하여 게시(publish)할 수 있다
가치: 검증된 데이터만 사용자에게 공개하고, 이전 버전으로 롤백할 수 있다
우선순위: High

인수 조건 (Acceptance Criteria):
  1. Given 데이터가 업로드되었을 때, When 버전 목록을 확인하면, Then 업로드 일시와 버전 번호가 표시되고 디폴트 상태는 '미리보기(preview)'이다
  2. Given 미리보기 상태의 버전이 있을 때, When '게시(publish)'를 실행하면, Then 해당 버전의 데이터가 FO에 표시된다
  3. Given 게시 상태의 버전이 있을 때, When 다른 버전을 게시하면, Then 이전 버전은 미리보기 상태로 변경되고 새 버전이 게시된다
  4. Given 게시 상태를 변경했을 때, When 저장하지 않고 페이지를 이탈하면, Then 변경사항 저장 확인 알림이 표시된다
  5. Given 버전 목록에서, When 이전 버전을 선택하여 게시하면, Then 해당 버전으로 FO 데이터가 롤백된다

  2. Given 캘린더 UI가 표시될 때, When 날짜를 클릭하면, Then 선택된 날짜가 입력 필드에 반영된다

  5. Given 동일한 날짜에 시점이 이미 존재할 때, When 같은 날짜로 추가를 시도하면, Then 중복 경고가 표시된다
  6. Given 시점이 등록되었을 때, When FO 그래프를 확인하면, Then 해당 날짜에 시점 마커가 표시된다
  7. Given 등록된 시점이 있을 때, When 수정 또는 삭제를 실행하면, Then 시점 정보가 업데이트/제거된다

유저 스토리 ID: US-A-2-1-01
기능: Feature A-2-1: [CMS] 엑셀 구조 매핑 관리
제목: 엑셀 구조 매핑 등록
역할: 여론조사 관리자 (박데이터)
목표: 업로드할 엑셀 파일의 시트명과 컬럼명을 DB 테이블 컬럼과 매핑할 수 있다
가치: 선거마다 다른 엑셀 구조를 사용하더라도 시스템이 올바르게 데이터를 파싱할 수 있다
우선순위: High

인수 조건 (Acceptance Criteria):
  1. Given 엑셀 매핑 관리 페이지에서, When 카테고리(정당지지율/후보자지지율)를 선택하면, Then 해당 카테고리의 매핑 설정 화면이 표시된다
  2. Given 매핑 설정 화면에서, When 시트명(예: "전체", "성", "연령대")과 필터타입(예: "ALL", "GENDER", "AGE")을 매핑하면, Then 시트명→필터타입 매핑이 저장된다
  3. Given 매핑 설정 화면에서, When 컬럼명(예: "party1", "candi1")과 DB 컬럼명을 매핑하면, Then 컬럼명→DB 컬럼 매핑이 저장된다
  4. Given 시트명 매핑에서, When 동일한 시트명이 중복 입력되면, Then 중복 경고가 표시된다
  5. Given 컬럼명 매핑에서, When 필수 컬럼(date)이 누락되면, Then 필수 컬럼 누락 경고가 표시된다
  6. Given 매핑이 저장되었을 때, When 엑셀 파일을 업로드하면, Then 저장된 매핑 규칙에 따라 데이터가 파싱된다
  7. Given 선거 A의 매핑이 설정되었을 때, When 선거 B로 전환하면, Then 선거 B의 매핑 설정이 별도로 관리된다

유저 스토리 ID: US-A-2-1-02
기능: Feature A-2-1: [CMS] 엑셀 구조 매핑 관리
제목: 엑셀 구조 매핑 조회 및 수정
역할: 여론조사 관리자 (박데이터)
목표: 등록된 엑셀 구조 매핑을 조회하고 수정할 수 있다
가치: 엑셀 구조가 변경되었을 때 매핑 정보를 업데이트할 수 있다
우선순위: High

인수 조건 (Acceptance Criteria):
  1. Given 엑셀 매핑 관리 페이지에서, When 카테고리를 선택하면, Then 해당 카테고리의 기존 매핑 목록이 표시된다
